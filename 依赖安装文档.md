## 1. 一键流程速览（macOS / zsh）

```bash

cd ..
```

## 2. 后端依赖（Python）

### 2.1 创建并激活虚拟环境

虚拟环境建在**项目根目录**，命名 `.venv`（不叫 `.env`，避免与 dotenv 配置文件混淆）。

```bash
# macOS / Linux (zsh/bash)
cd /Users/lme/study/project/工程实践
python3 -m venv .venv
source .venv/bin/activate          # 退出用 deactivate
```

```powershell
# Windows PowerShell
cd <仓库路径>\工程实践
py -m venv .venv
.venv\Scripts\Activate.ps1
```

激活后命令行前缀会出现 `(.venv)`。后续 `pip` / `python` / `flask` 均指向该环境。
> 不想激活也可直接用绝对路径调用：`.venv/bin/python`、`.venv/bin/pip`。

### 2.2 安装依赖

```bash
pip install -r backend/requirements.txt
```

国内网络慢时用镜像加速（清华源）：

```bash
pip install -r backend/requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 2.3 依赖清单（`backend/requirements.txt`）

| 包 | 约束 | 实测版本 | 用途 |
|----|------|---------|------|
| Flask | ==3.0.3 | 3.0.3 | Web 框架（应用工厂 + 蓝图） |
| Flask-Cors | ==4.0.1 | 4.0.1 | 跨域，仅放开 `/api/*` |
| Flask-SQLAlchemy | ==3.1.1 | 3.1.1 | ORM 集成 |

| python-dotenv | ==1.0.1 | 1.0.1 | 读取 `.env` 配置 |
| PyMySQL | ==1.1.1 | 1.1.1 | **MySQL 驱动**（默认；纯 Python，免编译） |
| numpy | >=1.26 | 2.0.2 | 数值计算（预测特征） |
| pandas | >=2.0 | 2.3.3 | 数据处理 |
| scikit-learn | >=1.3 | 1.6.1 | 随机森林房价预测（缺失时启发式降级） |
| requests | ==2.32.3 | 2.32.3 | 采集骨架的 HTTP 请求 |

> 全国大屏的后端数据服务（`services/national.py`）只用到 Python 标准库（`csv`），**无额外依赖**。

### 2.4 选择数据库驱动

- **MySQL（默认）**：`PyMySQL` 已在 requirements 中，无需额外操作。
- **PostgreSQL**：在 `backend/requirements.txt` 中**取消注释** `psycopg2-binary==2.9.9` 后重装，或单独安装：
  ```bash
  pip install psycopg2-binary==2.9.9
  ```
- **SQLite（零配置）**：无需任何驱动，Python 内置。

### 2.5 环境变量 `.env`

```bash
cp backend/.env.example backend/.env
```

关键项（详见 `.env.example` 注释）：

- `DATABASE_URL`——数据库连接串。
  - 当前项目默认 MySQL：`mysql+pymysql://用户:密码@127.0.0.1:3306/housing?charset=utf8mb4`（需先 `CREATE DATABASE housing DEFAULT CHARACTER SET utf8mb4;`）
  - **留空 / 删除该行 → 自动回退到本地 SQLite（`backend/housing.db`），无需任何数据库服务**。这适合临时演示，不是当前数据口径。
- `CORS_ORIGINS`——允许的前端来源，默认含 `http://localhost:5173`。
- `HOST` / `PORT`——后端监听地址，默认 `127.0.0.1:5000`。

### 2.6 初始化数据库与当前数据

建表在应用启动时自动完成。当前项目数据在 MySQL `housing` 中，来源主要是山东 `house_info.tsv` 与链家快照。若需要重新导入山东原始数据：

```bash
cd backend
env PYTHONDONTWRITEBYTECODE=1 ../.venv/bin/python import_shandong_house_info.py --file data/raw/house_info.tsv --replace
env PYTHONDONTWRITEBYTECODE=1 ../.venv/bin/python import_shandong_facilities.py --file data/raw/house_info.tsv --replace
cd ..
```

如只是做合成演示，可运行：

```bash
cd backend
../.venv/bin/python -m flask --app app seed
cd ..
```

> 当前正式演示不建议跑 seed。全国大屏真实接口读取数据库；`backend/data/province_data.csv` 的参考挂牌量接口仍保留备用。

### 2.7 爬虫依赖（Scrapy，可选）

如需运行 `crawler/`（链家二手房爬虫，详见 `crawler/README.md`），把 Scrapy 装进**同一个 `.venv`**（这样爬虫管道才能复用后端模型写库）：

```bash
source .venv/bin/activate
pip install -r crawler/requirements.txt     # 即 Scrapy>=2.11；itemadapter/parsel/lxml/twisted 等随之安装
```

> 爬虫写库复用 `backend` 的 Flask + SQLAlchemy，无需额外数据库依赖。实测 Scrapy 2.13.4 / Python 3.9.6 可用。

## 4. 特殊依赖与网络说明（重要）

### 4.1 `d3-geo`：未安装，已用自实现替代 ✅

大屏地图的**墨卡托投影**本可用 `d3-geo`，但本机环境 **npm 无法访问外网**（`npm install` 报网络错误）。为此项目内置了等价实现：

- `frontend/src/utils/projection.js`——自实现 `geoMercator().fitExtent()`，数学与 `d3-geo` 一致。
- **结论：无需安装 `d3-geo`，前端未新增任何 npm 依赖**，`npm install` 用现有 `package.json` 即可。

> 如果你的网络可用、且更想用官方库：`cd frontend && npm i d3-geo@^3.1.1`，再把 `GeoMap3D.vue` 顶部的 `import { geoMercator } from '@/utils/projection'` 改回 `from 'd3-geo'` 即可（可选，非必需）。

### 4.2 运行时 CDN 依赖：地图边界 GeoJSON（浏览器需联网）

下钻地图的行政边界在**运行时**从 DataV 拉取：
`https://geo.datav.aliyun.com/areas_v3/bound/{adcode}_full.json`（`100000`=全国）。

- 这不是安装期依赖，而是**浏览器运行期**的网络请求——浏览器需能访问该 CDN，地图才会显示。
- 离线兜底：可改用 `资料/static/data/` 与 `资料/other/` 下已有的本地 GeoJSON（需要时告知，我可加本地加载兜底）。

### 4.3 镜像源小抄

| 工具 | 命令 |
|------|------|
| pip（清华） | `pip install -r backend/requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple` |
| pip（永久） | `pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple` |
| npm（淘宝） | `npm config set registry https://registry.npmmirror.com` |
| npm（恢复官方） | `npm config set registry https://registry.npmjs.org` |

## 7. 常见问题（FAQ）

- **`numpy` / `scikit-learn` 安装失败（编译报错）**：先升级 pip（`python -m pip install -U pip`），改用镜像源装预编译 wheel；确认 Python 为 64 位、版本 ≥ 3.9。
- **MySQL 连接报错（如 `Access denied` / `Can't connect`）**：检查 `DATABASE_URL` 用户名/密码/端口、数据库 `housing` 是否已创建、`PyMySQL` 是否安装。临时跳过：把 `.env` 里 `DATABASE_URL` 留空，自动用 SQLite。
- **`mysqladmin ping` 提示 `Access denied for user ... (using password: NO)`**：MySQL 服务已经响应了，只是当前命令没带密码。用 `mysqladmin -u root -p ping` 或 `mysql -u root -p` 输入密码登录。
- **端口被占用**：后端 `5000`、前端 `5173`。后端改 `.env` 的 `PORT`；前端改 `frontend/vite.config.js` 的 `server.port`。
- **前端请求 `/api` 404 或跨域**：确认后端在 `:5000` 运行；`vite.config.js` 的代理与后端 `CORS_ORIGINS` 是否匹配前端地址。
- **大屏地图空白 / 区域不显示**：多为浏览器无法访问 DataV CDN（见 §4.2）；检查网络或改用本地 GeoJSON 兜底。
- **`flask --app app seed` 报 `No module named 'config'`**：必须在 `backend/` 目录内执行（后端用的是相对该目录的导入）。
- **`.venv/bin/flask` 报 `bad interpreter` 且路径还是旧电脑/旧目录**：这是虚拟环境脚本里记录了旧 Python 路径。优先使用 `../.venv/bin/python -m flask --app app ...`；若仍异常，重新创建 `.venv` 并重装依赖。
- **macOS 下载/拷贝后运行被拦截**：可在项目根目录执行 `xattr -dr com.apple.quarantine .venv frontend/node_modules` 清理隔离属性。
- **`npm install` 慢或失败**：切淘宝镜像（§4.3）；本项目未新增需联网拉取的特殊依赖，常规安装即可。
