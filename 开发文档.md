## 1. 项目概述

**项目名**：智慧房源探索平台（Smart Property Exploration Platform）
**性质**：工程实践课程项目（学习/演示用途，非生产系统）。

**目标**：采集二手房数据 → 分析与预测房价 → 以可视化（含 3D）方式呈现各城市/区域的房源与价格分布，辅助购房、租房、投资与市场了解；并展示房屋周边配套（学校、医院、地铁等）以评估生活便利性。

**新增方向（2026-06-03）**：在原平台基础上新增 **全国二手房数据可视化大屏**（深色科技风），以 **Three.js + 墨卡托投影**将 GeoJSON 边界挤出为立体地图，支持 **全国 → 省 → 市 三级下钻**（点击省份进入省级地图、点击城市进入市级地图），区域颜色映射二手房挂牌量。大屏作为**新首页/入口**（路由 `/`），原有页面（房源总览 `/dashboard`、探索、分析）保留并可经导航访问。

**数据方向更新（2026-06-16）**：开发库切换到 MySQL `housing`，并导入山东 `house_info.tsv` 大规模真实房源数据；当前 MySQL 中有 **126002 条房源 / 139 城 / 2488 个区或商圈**，其中 `shandong_house_info` 121617 条、`lianjia` 4385 条。大屏真实接口优先读取数据库，山东市级下钻会把商圈数据按行政区汇总展示。详见 §9、§14、§15。

## 3. 系统架构

```
浏览器 (Vue3 SPA, :5173)
   │  axios 请求 /api/*
   │  （开发期 Vite 代理 /api → :5000；生产可由 Nginx/网关转发）
   ▼
Flask 后端 (:5000)
   ├── api/        蓝图路由层（参数校验、组装响应）
   ├── services/   业务逻辑层（统计分析、ML 预测）
   ├── models/     ORM 数据模型层
   └── spider/     数据采集（骨架）
   ▼
数据库 (MySQL / PostgreSQL / SQLite)
```

- 响应统一信封：`{ "code": 0, "message": "ok", "data": ... }`；前端拦截器自动解包 `data`。
- 分层原则：路由层不写复杂逻辑，统计/预测下沉到 `services/`。

## 5. 数据库设计

共 5 张表，关系：City 1—N District 1—N Property；District 1—N Facility；District 1—N PriceHistory。

| 表 | 字段（关键） | 说明 |
|----|------------|------|
| `cities` 城市 | id, name, name_en, province, lng, lat | |
| `districts` 行政区 | id, **city_id**, name, lng, lat, grid_x, grid_y | grid_x/y 供 3D 网格布局 |
| `properties` 房源 | id, **district_id**, title, total_price(万), unit_price(元/㎡), area, rooms, halls, floor, total_floors, build_year, orientation, decoration, has_elevator, listing_type, lng, lat, source, source_url, created_at | |
| `facilities` 配套 | id, **district_id**, name, category(school/hospital/subway/mall/park), lng, lat | |
| `price_history` 价格走势 | id, **district_id**, month(YYYY-MM), avg_unit_price | 按区按月均价 |

## 9. 核心功能实现说明

**3D 房价地图（`CityMap3D.vue`）**：Scene/PerspectiveCamera/WebGLRenderer + OrbitControls；每个行政区按 `grid_x/grid_y` 排成网格，渲染为立体柱，**柱高与颜色映射区域均价**（teal→red 色阶）；Raycaster 实现悬停高亮 + HTML tooltip；点击 emit `select` 跳转到该区房源列表；含自动旋转（交互后停止）、自适应 resize、卸载时释放资源（dispose）。

**房价预测（`services/prediction.py`）**：特征 = 面积/室/厅/房龄/电梯/楼层比/区域均价；首次调用时按请求的 `district_id` 只取该行政区/商圈样本惰性训练 RandomForest 并缓存。若该区域样本少于 30 条或本机无 sklearn，则降级为该区域均值 + 房源特征修正的启发式公式；返回值包含 `method`、`training_scope`、`training_sample_count`，前端会展示模型口径。

**数据分析（`services/analysis.py`）**：SQL 聚合实现总览、各区均价排行、单价分布直方图；走势接口仍保留并读取 `price_history`，但当前真实库暂无连续月度历史数据，前端显示空态。

**投资热点分析（`analysis.investment_ranking` + `InvestmentChart.vue`）**：由于当前真实库没有可靠的连续历史涨幅，评分改为当前快照可支撑的五项指标：价格洼地 35%、市场热度/样本可靠性 25%、周边配套 20%、交易安全 10%、挂牌新鲜度 10%。价格洼地按城市内均价相对位置计算；市场热度用区/商圈房源套数计算；周边配套用设施数量与类别覆盖度计算；交易安全从 `quanshu/chanquan/diya/maidian` 中聚合商品房、非共有、无抵押、满二/满五等信息；挂牌新鲜度按该城市最新挂牌日期向前 12 个月的占比计算。前端散点图 x=均价、y=热度/配套指数、气泡大小=房源样本数、颜色=投资评分。

**山东房源扩展详情（`services/property_details.py` + `PropertyDetail.vue`）**：不改 `properties` 表结构，而是按房源 `source_url` 从 `backend/data/raw/house_info.tsv` 查询 `shijian/quanshu/chanquan/diya/maidian/jieshao/huxingjieshao/jiaotong`，详情页左下模块由“近 24 个月房价走势”改为“交易属性”，展示挂牌时间、交易权属、产权情况、抵押信息、核心卖点、小区介绍、户型介绍、交通出行。非山东或缺少原始链接的房源显示空态。

**周边配套抽取（`import_shandong_facilities.py`）**：从山东 TSV 的 `maidian/jieshao/huxingjieshao/jiaotong` 文本中按关键词抽取学校、医院、地铁、交通、商场、公园等配套标签，并按城市+区/商圈汇总写入 `facilities` 表。当前已生成 4513 条区/商圈级设施标签；它们不是逐房源 POI 坐标，而是用于详情页和投资评分的配套摘要。

**采集骨架（`spider/`）**：`BaseSpider`(礼貌限速请求 + 解析 + 入库)；`ExampleSpider` 解析内置示例 HTML 演示流程；含 robots.txt/ToS 合规提示。

**全国二手房数据大屏（`views/Screen.vue` + `components/screen/GeoMap3D.vue`）**：参考 `资料/sc-datav-main`（React + R3F 大屏，`public/demo_2.jpg` 风格）的**技法**，移植到本项目 Vue3 + Three.js（不复用其 React 组件）。

- **3D 下钻地图（`GeoMap3D.vue`）**：从 DataV GeoatlasGeoJSON 拉取行政边界（`geo.datav.aliyun.com/areas_v3/bound/{adcode}_full.json`，`100000`=全国，带缓存）；用**墨卡托投影**（`utils/projection.js`，自实现、与 d3-geo `geoMercator().fitExtent` 数学一致）将经纬度投影到平面，构造 `THREE.Shape`（含孔洞）→ `ExtrudeGeometry` 挤出立体；顶/侧分材质、顶部发光描边、质心处贴名称 Sprite。Raycaster 悬停高亮 + 抬升动画 + HTML tooltip；点击有下级（`childrenNum>0`）的区域即**下钻**（全国→省→市），`back()`/`goToLevel(i)` 返回上级；区域顶面颜色按 `dataMap` 数值（二手房挂牌量）线性映射蓝→青色阶，无数据走中性色。透明渲染露出大屏背景。通过 `levelchange` 事件向页面回传当前层级 `path`。
- **数据联动（`Screen.vue`）**：监听 `levelchange`——全国层调 `/api/national/real/provinces`、省层调 `/api/national/real/cities?province=`、市层调 `/api/national/real/districts?city=`，结果既驱动地图着色（`dataMap` 同时保存原始名与规范化名 → **真实房源数**）又驱动右侧排行/明细面板；KPI 取 `/real/summary`（真实房源数/覆盖省市商圈/均价/户型分布）。地图标签和 tooltip 同时显示区域名与房源套数，颜色匹配会同时尝试原始名、规范化名和去后缀名。
  - **山东市级下钻（2026-06-16 调整）**：`house_info.tsv` 入库时区字段主要是商圈/板块（`quyu`），直接展示会出现“青岛只有胶州”等行政区缺失错觉。后端 `real_districts(city)` 现在会读取原始 TSV 中 `region/quyu` 的对应关系，把商圈房源数汇总到行政区，再返回给市级下钻；名称规范化支持 `区/县/市` 后缀，且对 `单县`、`曹县` 等短县名做保护，避免截成单字。
  - **数据源说明（2026-06-16 调整）**：大屏由原先的 `资料/` 全国挂牌量参考数据、后来的 4347 条链家快照，升级为当前 MySQL 真实库驱动（126002 条，单位"套"），与 `/dashboard` 等内页数据一致；原 CSV 接口（`/api/national/{summary,provinces,cities}`）保留备用。
- **大屏布局**：`useAutoFit` 将 1920×1080 设计稿等比缩放铺满视口；三栏（左 KPI/结构环图/排行，中 面包屑+地图+提示，右 城市房源数 TOP10/均价 TOP/数据明细），`ScreenPanel` 提供带四角描边的深色面板。大屏为独立全屏路由 `/`，`App.vue` 在该路由隐藏通用页头/页脚，大屏自带返回主站导航。
- **离线适配**：因环境无法访问 npm 安装 `d3-geo`，改为自实现等价墨卡托投影（`utils/projection.js`），零外部新增依赖。地图 GeoJSON 运行时从 DataV CDN 拉取（需联网）。

## 13. 已知问题与风险

- 构建产物中 Three.js + ECharts 的 vendor chunk > 500KB（仅告警，不影响运行）；后续按需加载优化。
- 预测模型为演示级（特征以区域均价为主导），精度有限，仅作教学演示。
- `seed.py` 数据为合成演示数据，价格水平为示意；当前正式演示不建议运行 seed，默认使用 MySQL 中已导入的真实库。
- 真实爬虫（`crawler/`，Scrapy）需遵守目标站 robots.txt 与服务条款；已内置限速、抓取安全阀、不绕过反爬。真实站点有反爬，抓取可能被限流或返回验证页，请勿试图绕过。
- 非山东链家数据仍主要是**单次首页快照**（每城约 30 条、分散到数十个商圈），单个商圈均价样本量小、波动较大；山东数据量更大，但原始字段以商圈为主，行政区展示依赖 `house_info.tsv` 的 `region/quyu` 映射。
- 当前真实库仍缺少完整历史走势与真实 POI 坐标数据；房价走势保留空态，周边配套为从文本抽取的区/商圈级标签，不能代表精确距离或坐标。
- 投资热点为当前快照下的相对评分，适合做区域比较，不等同于真实收益率或未来涨幅预测。
- SQLite 旧库仍保留为 `backend/housing.db` / `backend/housing.db.bak`；当前 `.env` 指向 MySQL。若误运行旧虚拟环境里的 `flask` 脚本出现 `bad interpreter`，使用 `../.venv/bin/python -m flask --app app ...` 或重建虚拟环境脚本。
